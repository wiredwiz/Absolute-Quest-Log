AbsoluteQuestLog-1.0 Changelog
==============================

Version 2.2.8 (March 2026)
---------------------------
- Bug fix: increased debounce timer from 50ms to 500ms. The two
  post-placement QUEST_LOG_UPDATE events are separated by a server
  round-trip (~400ms max). The 50ms window fired between them, processing
  the intermediate low count. 500ms batches both events into one rebuild
  against the settled state.


Version 2.2.7 (March 2026)
---------------------------
- Bug fix: replaced cursor-detection logic with a debounce on every
  QUEST_LOG_UPDATE. Each call schedules a 50ms C_Timer; only the latest
  timer fires the rebuild. Bag operations produce 2 rapid events (one
  with intermediate low count, one correct) — both are swallowed, and
  one settled rebuild runs after the quiet window. Cursor detection
  failed because CursorHasItem() returns false throughout certain
  combine operations, so the guard never activated.


Version 2.2.6 (March 2026)
---------------------------
- Bug fix: replaced skip-one-event with a 500 ms settle timer. After the cursor
  empties, cursorHadItem stays true (suppressing all intermediate events) until
  the timer fires. The timer callback clears the flag and rebuilds against the
  fully settled bag state. Handles cases where WoW fires two post-placement
  events (one with intermediate low count, one with correct count). A
  bagSettleTimerPending guard prevents duplicate timers.


Version 2.2.5 (March 2026)
---------------------------
- Bug fix: false regression/progression still fired after 2.2.4. The
  C_Timer.After(0) approach fired too early — the timer runs at the end of Frame
  N, but the bag-settle QUEST_LOG_UPDATE arrives on Frame N+1, so the timer
  rebuild always read the intermediate count. Fix: skip exactly the first
  QUEST_LOG_UPDATE after cursor empties and let the next natural event rebuild
  with the correct settled count. Affects all placement types (empty slot and
  existing stack).


Version 2.2.4 (March 2026)
---------------------------
- Bug fix: false regression/progression when dropping split quest items into an
  empty bag slot. WoW fires QUEST_LOG_UPDATE after cursor empties but before the
  destination slot is counted. Added one-frame settle defer (C_Timer.After(0))
  triggered by the cursor-empty transition, so the rebuild always sees the final
  correct count. Placing into an existing stack is unaffected.


Version 2.2.3 (March 2026)
---------------------------
- Bug fix: handleQuestLogUpdate() now guards against QUEST_LOG_UPDATE events fired
  while the player has an item on the cursor (e.g., splitting or moving bag stacks
  of quest collectibles). The objective count during these operations represents an
  unstable intermediate state. AQL defers the cache rebuild until the cursor is empty;
  the next event sees a net-zero diff and fires no callbacks. Eliminates false
  AQL_OBJECTIVE_REGRESSED, AQL_OBJECTIVE_PROGRESSED, and AQL_OBJECTIVE_COMPLETED
  events caused by bag operations.


Version 2.2.2 (March 2026)
---------------------------
- Added addon logo for addon screen


Version 2.2.1 (March 2026)
---------------------------
- Added AQL.Event constant table for all 12 AQL callback event strings.
  Consumers should now register callbacks using the constants:
    AQL:RegisterCallback(AQL.Event.QuestAccepted, handler, target)
  Constants: QuestAccepted, QuestAbandoned, QuestCompleted, QuestFinished,
  QuestFailed, QuestTracked, QuestUntracked, ObjectiveProgressed,
  ObjectiveCompleted, ObjectiveRegressed, ObjectiveFailed, UnitQuestLogChanged.

- Replaced GetQuestLogZoneNames() with GetQuestLogZones(). The new method
  returns an array of {name, isCollapsed} tables instead of plain strings,
  making it easy to capture and restore collapsed state around bulk operations:
    local zones = AQL:GetQuestLogZones()
    AQL:ExpandAllQuestLogHeaders()
    -- ... do work ...
    for _, z in ipairs(zones) do
        if z.isCollapsed then AQL:CollapseQuestLogZoneByName(z.name) end
    end

- Added doc comments to all Group 1 public API methods.


Version 2.2.0 (March 2026)
---------------------------
- Added Quest Log Frame & Navigation wrappers to WowQuestAPI: GetNumQuestLogEntries,
  GetQuestLogTitle, GetQuestLogSelection, SelectQuestLogEntry, GetQuestLogPushable,
  QuestLog_SetSelection, QuestLog_Update, ExpandQuestHeader, CollapseQuestHeader,
  ShowQuestLog, HideQuestLog, IsQuestLogShown, GetQuestDifficultyColor, GetPlayerLevel.

- Added Group 2: Quest Log APIs to the AQL public interface.
  Thin wrappers (one-to-one with WoW globals):
    ShowQuestLog, HideQuestLog, IsQuestLogShown, GetQuestLogSelection,
    SelectQuestLogEntry, IsQuestLogShareable, SetQuestLogSelection,
    ExpandQuestLogHeader, CollapseQuestLogHeader, GetQuestDifficultyColor,
    GetQuestLogIndex.
  Compound ByIndex (multi-step operations):
    IsQuestIndexShareable, SelectAndShowQuestLogEntryByIndex, OpenQuestLogByIndex,
    ToggleQuestLogByIndex, GetSelectedQuestId, GetQuestLogEntries, GetQuestLogZones,
    ExpandAllQuestLogHeaders, CollapseAllQuestLogHeaders, ExpandQuestLogZoneByName,
    CollapseQuestLogZoneByName, ToggleQuestLogZoneByName, IsQuestLogZoneCollapsed.
  Compound ById (resolve questID to logIndex, then delegate to ByIndex):
    IsQuestIdShareable, SelectAndShowQuestLogEntryById, OpenQuestLogById,
    ToggleQuestLogById.

- Added Group 1: Player & Level section.
    GetPlayerLevel, GetQuestsInQuestLogBelowLevel, GetQuestsInQuestLogAboveLevel,
    GetQuestsInQuestLogBetweenLevels, GetQuestsInQuestLogBelowLevelDelta,
    GetQuestsInQuestLogAboveLevelDelta, GetQuestsInQuestLogWithinLevelRange.

- Reorganized AbsoluteQuestLog.lua public API into two groups with subsections:
  Group 1 (Quest APIs): Quest State, Quest History, Quest Resolution, Objectives,
  Chain Info, Quest Tracking, Player & Level.
  Group 2 (Quest Log APIs): Thin Wrappers, Compound ByIndex, Compound ById.


Version 2.1.1 (March 2026)
---------------------------
